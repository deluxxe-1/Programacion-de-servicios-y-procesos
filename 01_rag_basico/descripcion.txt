Este proyecto implementa un sistema RAG básico (Retrieval-Augmented Generation) que permite realizar búsquedas semánticas sobre documentos de texto. La idea fundamental es convertir texto en representaciones numéricas llamadas embeddings, almacenarlas en una base de datos vectorial y luego recuperar los fragmentos más relevantes ante una consulta del usuario, todo ello basándose en similitud de significado y no en coincidencia literal de palabras.

El flujo de uso del programa se estructura en dos etapas. Primero, se ejecuta la fase de indexación, donde el sistema lee los documentos de texto, los divide en fragmentos manejables mediante un proceso de chunking, genera el embedding de cada fragmento utilizando el modelo nomic-embed-text y almacena estos vectores en ChromaDB. Esta fase solo necesita ejecutarse una vez por cada conjunto de documentos, ya que la base de datos persiste entre sesiones.

Una vez completada la indexación, el usuario puede realizar consultas en lenguaje natural. El sistema convierte la pregunta en un embedding utilizando el mismo modelo, busca en ChromaDB los fragmentos cuyos vectores sean más similares mediante distancia coseno, y devuelve los chunks más relevantes en su forma cruda. Es importante señalar que en esta versión básica no hay procesamiento adicional de los resultados: se muestran directamente los fragmentos recuperados tal cual están almacenados.

Desde el punto de vista tecnológico, el proyecto utiliza Python como lenguaje de desarrollo. Los embeddings se generan con el modelo nomic-embed-text de Ollama, que produce vectores de 768 dimensiones capaces de capturar el significado semántico del texto. ChromaDB actúa como base de datos vectorial, ofreciendo almacenamiento persistente y búsqueda eficiente por similitud coseno.

La elección de ChromaDB responde a su facilidad de integración con Python y a que no requiere infraestructura adicional, ya que puede funcionar de forma embebida directamente en la aplicación. El modelo nomic-embed-text, por su parte, se ejecuta localmente a través de Ollama, manteniendo la coherencia con el enfoque de privacidad local que comparten todos los proyectos.

Este ejercicio sienta las bases para entender cómo funciona internamente un sistema RAG y constituye el punto de partida sobre el que se construyen las versiones más avanzadas desarrolladas en los siguientes ejercicios.
